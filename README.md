<h2><a id="user-content-deep-learning-project" class="anchor" aria-hidden="true" href="#deep-learning-project"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Deep Learning Project</h2>
<p>In this project, you will train a deep neural network to identify and track a target in simulation. So-called “follow me” applications like this are key to many fields of robotics and the very same techniques you apply here could be extended to scenarios like advanced cruise control in autonomous vehicles or human-robot collaboration in industry.</p>

<h2><a id="user-content-setup-instructions" class="anchor" aria-hidden="true" href="#setup-instructions"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Setup Instructions</h2>
<p><strong>Clone the repository</strong></p>
<pre><code>$ git clone https://github.com/udacity/RoboND-DeepLearning.git
</code></pre>
<p><strong>Download the data</strong></p>
<p>Save the following three files into the data folder of the cloned repository.</p>
<p><a href="https://s3-us-west-1.amazonaws.com/udacity-robotics/Deep+Learning+Data/Lab/train.zip" rel="nofollow">Training Data</a></p>
<p><a href="https://s3-us-west-1.amazonaws.com/udacity-robotics/Deep+Learning+Data/Lab/validation.zip" rel="nofollow">Validation Data</a></p>

<p><strong>Download the QuadSim binary</strong></p>
<p>To interface your neural net with the QuadSim simulator, you must use a version QuadSim that has been custom tailored for this project. The previous version that you might have used for the Controls lab will not work.</p>
<p>The simulator binary can be downloaded <a href="https://github.com/udacity/RoboND-DeepLearning/releases/latest">here</a></p>
<p><strong>Install Dependencies</strong></p>
<p>You'll need Python 3 and Jupyter Notebooks installed to do this project.  The best way to get setup with these if you are not already is to use Anaconda following along with the <a href="https://github.com/udacity/RoboND-Python-StarterKit">RoboND-Python-Starterkit</a>.</p>

<li>SciPy 0.17.0</li>

<h2><a id="user-content-collecting-training-data" class="anchor" aria-hidden="true" href="#collecting-training-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Collecting Training Data</h2>

data/train/images - contains images for the training set
data/train/masks - contains masked (labeled) images for the training set
data/validation/images - contains images for the validation set
data/validation/masks - contains masked (labeled) images for the validation set
data/weights - contains trained TensorFlow models

data/raw_sim_data/train/run1
data/raw_sim_data/validation/run1
</code></pre>
<h3><a id="user-content-training-set" class="anchor" aria-hidden="true" href="#training-set"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Training Set</h3>
<ol>
<li>Run QuadSim</li>
<li>Click the <code>DL Training</code> button</li>
<li>Set patrol points, path points, and spawn points. <strong>TODO</strong> add link to data collection doc</li>

<li>When you have finished collecting data, hit "r" to stop recording.</li>

<h3><a id="user-content-validation-set" class="anchor" aria-hidden="true" href="#validation-set"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Validation Set</h3>
<p>To collect the validation set, repeat both sets of steps above, except using the directory <code>data/raw_sim_data/validation</code> instead rather than <code>data/raw_sim_data/train</code>.</p>
<h3><a id="user-content-image-preprocessing" class="anchor" aria-hidden="true" href="#image-preprocessing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Image Preprocessing</h3>
<p>Before the network is trained, the images first need to be undergo a preprocessing step. The preprocessing step transforms the depth masks from the sim, into binary masks suitable for training a neural network. It also converts the images from .png to .jpeg to create a reduced sized dataset, suitable for uploading to AWS.
To run preprocessing:</p>
<pre><code>$ python preprocess_ims.py
</code></pre>
<p><strong>Note</strong>: If your data is stored as suggested in the steps above, this script should run without error.</p>
<p><strong>Important Note 1:</strong></p>
<p>Running <code>preprocess_ims.py</code> does <em>not</em> delete files in the processed_data folder. This means if you leave images in processed data and collect a new dataset, some of the data in processed_data will be overwritten some will be left as is. It is recommended to <strong>delete</strong> the train and validation folders inside processed_data(or the entire folder) before running <code>preprocess_ims.py</code> with a new set of collected data.</p>
<p><strong>Important Note 2:</strong></p>
<p>The notebook, and supporting code assume your data for training/validation is in data/train, and data/validation. After you run <code>preprocess_ims.py</code> you will have new <code>train</code>, and possibly <code>validation</code> folders in the <code>processed_ims</code>.
Rename or move <code>data/train</code>, and <code>data/validation</code>, then move <code>data/processed_ims/train</code>, into <code>data/</code>, and  <code>data/processed_ims/validation</code>also into <code>data/</code></p>
<p><strong>Important Note 3:</strong></p>
<p>Merging multiple <code>train</code> or <code>validation</code> may be difficult, it is recommended that data choices be determined by what you include in <code>raw_sim_data/train/run1</code> with possibly many different runs in the directory. You can create a tempory folder in <code>data/</code> and store raw run data you don't currently want to use, but that may be useful for later. Choose which <code>run_x</code> folders to include in <code>raw_sim_data/train</code>, and <code>raw_sim_data/validation</code>, then run  <code>preprocess_ims.py</code> from within the 'code/' directory to generate your new training and validation sets.</p>
<h2><a id="user-content-training-predicting-and-scoring" class="anchor" aria-hidden="true" href="#training-predicting-and-scoring"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Training, Predicting and Scoring</h2>
<p>With your training and validation data having been generated or downloaded from the above section of this repository, you are free to begin working with the neural net.</p>
<p><strong>Note</strong>: Training CNNs is a very compute-intensive process. If your system does not have a recent Nvidia graphics card, with <a href="https://developer.nvidia.com/cudnn" rel="nofollow">cuDNN</a> and <a href="https://developer.nvidia.com/cuda" rel="nofollow">CUDA</a> installed , you may need to perform the training step in the cloud. Instructions for using AWS to train your network in the cloud may be found <a href="https://classroom.udacity.com/nanodegrees/nd209/parts/09664d24-bdec-4e64-897a-d0f55e177f09/modules/cac27683-d5f4-40b4-82ce-d708de8f5373/lessons/197a058e-44f6-47df-8229-0ce633e0a2d0/concepts/27c73209-5d7b-4284-8315-c0e07a7cd87f?contentVersion=1.0.0&amp;contentLocale=en-us" rel="nofollow">here</a></p>
<h3><a id="user-content-training-your-model" class="anchor" aria-hidden="true" href="#training-your-model"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Training your Model</h3>
<p><strong>Prerequisites</strong></p>
<ul>
<li>Training data is in <code>data</code> directory</li>
<li>Validation data is in the <code>data</code> directory</li>
<li>The folders <code>data/train/images/</code>, <code>data/train/masks/</code>, <code>data/validation/images/</code>, and <code>data/validation/masks/</code> should exist and contain the appropriate data</li>
</ul>
<p>To train complete the network definition in the <code>model_training.ipynb</code> notebook and then run the training cell with appropriate hyperparameters selected.</p>
<p>After the training run has completed, your model will be stored in the <code>data/weights</code> directory as an <a href="https://en.wikipedia.org/wiki/Hierarchical_Data_Format" rel="nofollow">HDF5</a> file, and a configuration_weights file. As long as they are both in the same location, things should work.</p>
<p><strong>Important Note</strong> the <em>validation</em> directory is used to store data that will be used during training to produce the plots of the loss, and help determine when the network is overfitting your data.</p>
<p>The <strong>sample_evalution_data</strong> directory contains data specifically designed to test the networks performance on the FollowME task. In sample_evaluation data are three directories each generated using a different sampling method. The structure of these directories is exactly the same as <code>validation</code>, and <code>train</code> datasets provided to you. For instance <code>patrol_with_targ</code> contains an <code>images</code> and <code>masks</code> subdirectory. If you would like to the evaluation code on your <code>validation</code> data a copy of the it should be moved into <code>sample_evaluation_data</code>, and then the appropriate arguments changed to the function calls in the <code>model_training.ipynb</code> notebook.</p>
<p>The notebook has examples of how to evaulate your model once you finish training. Think about the sourcing methods, and how the information provided in the evaluation sections relates to the final score. Then try things out that seem like they may work.</p>
<h2><a id="user-content-scoring" class="anchor" aria-hidden="true" href="#scoring"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Scoring</h2>
<p>To score the network on the Follow Me task, two types of error are measured. First the intersection over the union for the pixelwise classifications is computed for the target channel.</p>
<p>In addition to this we determine whether the network detected the target person or not. If more then 3 pixels have probability greater then 0.5 of being the target person then this counts as the network guessing the target is in the image.</p>
<p>We determine whether the target is actually in the image by whether there are more then 3 pixels containing the target in the label mask.</p>
<p>Using the above the number of detection true_positives, false positives, false negatives are counted.</p>
<p><strong>How the Final score is Calculated</strong></p>
<p>The final score is the pixelwise <code>average_IoU*(n_true_positive/(n_true_positive+n_false_positive+n_false_negative))</code> on data similar to that provided in sample_evaulation_data</p>
<p><strong>Ideas for Improving your Score</strong></p>
<p>Collect more data from the sim. Look at the predictions think about what the network is getting wrong, then collect data to counteract this. Or improve your network architecture and hyperparameters.</p>
<p><strong>Obtaining a Leaderboard Score</strong></p>
<p>Share your scores in slack, and keep a tally in a pinned message. Scores should be computed on the sample_evaluation_data. This is for fun, your grade will be determined on unreleased data. If you use the sample_evaluation_data to train the network, it will result in inflated scores, and you will not be able to determine how your network will actually perform when evaluated to determine your grade.</p>
